# tests index

estructura principal:

- `tests/core/`: pruebas del nucleo del framework (`model`, `set_manager`, componentes base).
- `tests/blocks/`: pruebas unitarias por bloque economico.
- `tests/calibration/`: pruebas de calibracion PEP por fase.
- `tests/templates/`: pruebas de plantillas y paridad de templates.
- `tests/qa/`: pruebas de contratos, manifest y checks de calidad de datos.
- `tests/scripts/`: pruebas de pipelines y scripts operativos.
- `tests/babel/`: pruebas de lectura/escritura GDX y utilidades de datos.
- `tests/backends/`: pruebas de backends de modelado/solucion.
- `tests/fixtures/`: artefactos de prueba.
